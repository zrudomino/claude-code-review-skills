## Key Decisions Made
- **Ship A + B together**: Integrated both review-pipeline and fix-and-verify in one go instead of phased rollout.
  - **Context**: Strategist and Codex advised phased rollout (review-pipeline first, observe, then fix-and-verify). User overrode via AskUserQuestion.
  - **Rationale**: User preference; the canonical/convenience-copy pattern made both integrations cohesive and the risk of half-shipped state was higher than full ship.
- **Gate 8 placement between Step 4 and Step 5**: Fires after Codex Adversarial and before Commit.
  - **Rationale**: Codex's recommendation â€” tightest pre-commit guardrail, no wasted scans on losing candidates that fail earlier gates.
- **Scanner-native identity keys, never raw file+line**: Semgrep `match_based_id`, Gitleaks `Fingerprint`, OSV tuple `(ecosystem, package, advisory_id, manifest_path)`.
  - **Rationale**: Raw line numbers shift under unrelated edits above a finding, creating false positives for Gate 8 delta. AST-based fingerprints survive line shifts.
- **OSV identity tuple excludes version**: Dep bumps inside vulnerable range keep same finding identity.
  - **Rationale**: Prevents "trade CVE A for CVE B" churn from looking like a resolve + fresh finding in alternating rounds.
- **Two-phase tombstoning**: Absent in run N â†’ `sec_resolved_pending`; absent in run N+1 â†’ `fixed`.
  - **Rationale**: Skeptic mitigation â€” avoids noisy flip-flops when OSV data changes between runs.
- **Per-source quotas (min 8 agent / max 7 scan in 15-cap)**: Scan findings can't starve agents.
  - **Rationale**: Codex concern â€” a bulk Semgrep run could otherwise evict all agent findings from the cap.
- **OSV manifest-guard short-circuit**: Skip OSV at Step 5A if diff touches no dep manifest.
  - **Rationale**: Architect performance concern â€” OSV is slow and pointless when no lockfile changed.
- **Scan completeness is a first-class hard fail**: `complete=false` escalates, never silent-passes.
  - **Rationale**: Codex invariant â€” silent skip on broken infra is the worst failure mode for a security gate.
- **New reference files allowed (exactly 2)**: `security-corpus.md` + `security-gate-8.md`.
  - **Rationale**: skill-creator + skill-reviewer feedback. Earlier draft had "no new files" rule; paid for it with SKILL.md bloat + a forward reference that violated self-containment. Relaxed per progressive disclosure.
